# Tabs Session Saver Privacy Policy

Effective date: July 2026  
Last updated: July 2026

## 1. Overview

This Privacy Policy explains how the Tabs Session Saver Chrome Extension ("the Extension", "Tabs Session Saver", "we", "us") processes user data.

Tabs Session Saver is designed to work locally by default. Cloud Sync and newsletter signup are optional features. The Extension does not sell user data, use advertising, track users across websites, or use saved session data for profiling or analytics.

## 2. Local-First Operation

By default, saved sessions and preferences remain on the user's device.

Locally stored session data may include:

- tab titles;
- tab URLs;
- window structure;
- Chrome tab group names, colors, and tab membership;
- restore-related metadata;
- extension preferences.

This local data is stored with Chrome's `chrome.storage.local` API and remains accessible only to the Extension unless the user explicitly enables Cloud Sync or exports a JSON backup.

## 3. Optional Cloud Sync

Cloud Sync is optional. If the user chooses to enable Cloud Sync, the Extension uses Google Sign-In through Chrome's `chrome.identity` API.

When Cloud Sync is enabled, the following data may be processed:

- Google account profile data required for sign-in: Google user ID, email address, display name, and profile picture URL;
- saved session data: tab titles, URLs, window structure, tab groups, timestamps, and related session metadata;
- sync metadata: revision numbers, device identifiers generated by the Extension, timestamps, and sync status information.

Cloud Sync data is transmitted to the Tabs Session Saver sync backend hosted on Cloudflare Workers and stored in Cloudflare D1.

Cloud Sync is not enabled automatically. The user must explicitly sign in and start using Cloud Sync.

## 4. Google Sign-In

The Extension uses Google OAuth only to authenticate the user for Cloud Sync.

Requested OAuth scopes:

- `openid`
- `email`
- `profile`

These scopes are used only to identify the user's sync account. Tabs Session Saver does not access Google Drive, Gmail, Calendar, Contacts, or other Google account data.

Google OAuth access tokens are used to authenticate requests to the Tabs Session Saver sync backend. The Extension does not show these tokens to users and does not use them for purposes other than Cloud Sync authentication.

## 5. Newsletter Signup

If the user voluntarily subscribes to the newsletter, the Extension processes the email address entered by the user.

The email address is sent to a server-side proxy endpoint and then forwarded to the newsletter provider or automation service configured by the developer. Newsletter signup is optional and is not required to use the Extension.

The newsletter feature is used only to send product updates or related communications about Tabs Session Saver.

## 6. Static Assets

The Extension's welcome page may display static images, such as logos or screenshots, hosted on external public storage.

When these assets load, the browser may make standard web requests to the hosting provider. These requests do not include saved sessions, tab URLs, preferences, or user-generated session data.

## 7. Permissions and Purpose

Tabs Session Saver requests only permissions needed to provide its functionality:

| Permission | Purpose |
| --- | --- |
| `tabs` | Read tab titles, URLs, and tab state when saving sessions, and reopen tabs when restoring sessions. |
| `tabGroups` | Save and restore Chrome tab group information. |
| `windows` | Save and restore window-level information. |
| `storage` | Store sessions, preferences, and sync settings locally. |
| `alarms` | Schedule optional Auto Save behavior configured by the user. |
| `identity` | Let the user sign in with Google for optional Cloud Sync. |

The Extension does not use these permissions for background tracking, advertising, or profiling.

## 8. Third-Party Services

Tabs Session Saver may interact with the following services:

- Google OAuth: user authentication for optional Cloud Sync.
- Cloudflare Workers and Cloudflare D1: hosting and storage for optional Cloud Sync.
- Google Cloud Storage or equivalent static hosting: hosting non-interactive welcome page assets.
- Chrome Web Store: distribution and updates.
- Google Sites or the developer's website: hosting public legal documents.
- Newsletter provider or automation service: optional newsletter signup only.

No third-party service receives saved session data unless the user enables Cloud Sync or voluntarily submits newsletter information.

## 9. Security

Cloud Sync data is transmitted over HTTPS.

Cloud Sync data is stored on the sync backend so that the user can access it from multiple devices. The current Cloud Sync implementation is not end-to-end encrypted. This means the service operator may technically be able to access stored sync data when necessary for operating, debugging, securing, or maintaining the service.

Users who do not want session data to leave their device should keep Cloud Sync disabled.

## 10. Data Retention

Local data remains on the user's device until the user deletes it, imports over it, resets browser storage, or uninstalls the Extension.

Cloud Sync data remains stored while Cloud Sync is enabled or until the user requests deletion.

Newsletter email data remains stored until the user unsubscribes or requests deletion.

## 11. User Controls

Users can:

- save, rename, restore, preview, and delete sessions;
- remove individual tabs from saved sessions;
- export and import JSON backups;
- enable or disable Cloud Sync;
- push or pull Cloud Sync manually;
- uninstall the Extension to remove locally stored extension data;
- request deletion of Cloud Sync or newsletter data.

## 12. Children's Privacy

Tabs Session Saver is not intended for children under 13 and does not knowingly collect data from children.
